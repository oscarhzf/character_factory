# 01_MVP产品规格书 PRD

## 1. 项目背景
当前角色库搭建流程高度依赖人工：
- 手动复制 Prompt
- 手动多次生图
- 肉眼筛选
- 手工总结问题
- 手工改 Prompt
- 再次重复

该流程存在三个问题：
1. 低效：7 个角色需要大量重复劳动
2. 不稳定：Prompt 迭代缺少结构化沉淀
3. 不可追踪：无法系统复盘“为什么这一轮更好/更坏”

因此需要构建一个角色资产生产工作台，将流程工程化。

## 2. 产品定位
一个面向动漫角色库生产的内部 Web 工具。
它不是通用生图站，也不是纯 Prompt 面板，而是“角色资产生产流水线”。

## 3. 核心目标
### 3.1 MVP 目标
完成以下闭环：
1. 管理角色设定
2. 根据模板生成多个 Prompt 变体
3. 批量出候选图
4. 自动视觉评审并打标签
5. 多图对比与人工筛选
6. 自动生成下一轮 Prompt Patch
7. 锁定标准稿并沉淀版本历史

### 3.2 非目标
本阶段不做：
- 全自动无人值守定稿
- 复杂多智能体编排
- 视频生成
- 训练自有模型
- 团队协作权限系统
- 商业化收费逻辑

## 4. 目标用户
### 主用户
项目负责人 / 角色库设计管理者
### 次用户
后续可能接入的提示词优化人员、审稿人员

## 5. 核心业务对象
### 5.1 Universe（世界观）
统一的全局风格宪法：
- 4 头身
- 新国潮都市拟人
- 轻热血喜剧氛围
- sheet 风格统一
- 上色与线稿风格统一

### 5.2 Character（角色）
每个角色拥有：
- 角色代号
- 固定设定
- 半固定设定
- 可变出图参数
- 标准稿
- 历史版本

### 5.3 Generation Job（出图任务）
一次围绕单角色展开的候选图生成与评审任务。

### 5.4 Review（评审）
分为：
- 自动评审
- 人工评审

### 5.5 Master Image（标准稿）
人工最终确认的角色标准图，可作为后续精修或新任务的锚点。

## 6. 用户故事
### 6.1 角色管理员
- 作为角色管理员，我希望先配置角色固定设定，再基于模板发起批量出图任务。
- 作为角色管理员，我希望一次生成多个候选图，而不是单张碰运气。
- 作为角色管理员，我希望系统自动告诉我每张图哪里不合格。
- 作为角色管理员，我希望能快速对比多图并选出最优图。
- 作为角色管理员，我希望系统根据失败原因自动生成下一轮 Prompt Patch。
- 作为角色管理员，我希望把某张图锁定为标准稿，并在后续继续复用。

## 7. 关键流程
### 7.1 新角色探索流程
1. 创建角色
2. 配置世界观与角色固定设定
3. 配置任务描述
4. 系统编译 3 组 Prompt 变体
5. 每组生成 4 张，共 12 张
6. 系统自动评审并排序
7. 用户筛选 Top 2
8. 系统生成 Patch
9. 开启下一轮精修
10. 用户将最优图锁为标准稿

### 7.2 已有角色精修流程
1. 选择某角色的 Master 图或候选图
2. 创建 refine 任务
3. 只修局部问题
4. 自动评审
5. 用户确认是否替换 Master 图

## 8. 页面需求
### 8.1 角色库页
功能：
- 查看所有角色卡片
- 查看当前 Master 图
- 查看最近任务结果摘要
- 进入该角色工作台

角色卡片展示字段：
- 角色代号
- 名称
- 当前状态
- 当前标准稿缩略图
- 最近平均分
- 最近失败标签 Top 3
- 最近一次更新时间

### 8.2 角色详情 / 工作台页
功能：
- 查看角色宪法
- 查看 Prompt 模板
- 查看 Master 图
- 查看历史任务
- 创建探索任务 / 精修任务

### 8.3 创建任务页
表单字段：
- 角色
- 任务模式：explore / refine / edit
- 动作描述
- 道具
- 表情
- 视角
- 构图
- 锚点图
- 每组张数
- 变体组数
- 图片尺寸
- 质量档位

### 8.4 任务详情页
功能：
- 查看任务基础信息
- 查看 prompt 变体
- 查看候选图
- 查看自动评分与失败标签
- 查看 patch 建议
- 进入多图对比

### 8.5 多图对比页
功能：
- 2 图对比
- 4 图宫格对比
- 展示评分维度
- 展示失败标签
- 一键标记 keep / reject / candidate_master / set_master / refine_from_this

### 8.6 Prompt 版本页
功能：
- 查看 prompt 历史版本
- 查看 patch 差异
- 查看分数变化
- 查看 revised prompt
- 查看该版衍生出的图片

## 9. 自动评审指标
总分 100，推荐维度：
- 风格统一度：25
- 角色辨识度：20
- 4 头身准确度：20
- 动作与道具符合度：10
- 配色命中度：10
- Sheet 完整性：10
- 标准稿潜力：5

失败标签：
- ratio_too_tall
- legs_too_long
- head_too_small
- style_drift
- palette_off
- prop_missing
- expression_weak
- sheet_format_broken
- identity_not_distinct

## 10. 规则约束
### 10.1 必须
- 所有图片都必须绑定任务与 prompt 版本
- 所有自动评审都必须结构化落库
- 所有 Master 图必须可以回滚
- 所有自动 Patch 都必须可查看原文
- 用户人工判断优先于自动评分

### 10.2 禁止
- 页面直接硬编码 Prompt
- 只保存最终图片不保存中间评审数据
- 自动替换 Master 图而不经人工确认

## 11. 成功标准
### 11.1 功能验收
- 可管理 7 个角色
- 可批量生成候选图
- 可自动评审
- 可多图对比
- 可精修
- 可锁定标准稿

### 11.2 体验验收
- 单角色完成一轮探索任务不超过 1 个主要工作流页面跳转链
- 人工不需要再手工复制粘贴 Prompt 反复操作
- 用户可在一个页面看懂一轮任务的“输入 - 输出 - 评分 - 决策 - 下一步”

### 11.3 质量验收
- 同角色后续图像风格漂移下降
- 世界观风格统一性提升
- 失败类型可统计、可复盘、可归因
