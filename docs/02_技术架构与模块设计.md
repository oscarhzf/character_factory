# 02_技术架构与模块设计

## 1. 总体架构
建议采用 Monorepo。

```txt
apps/web                前端工作台
packages/core           领域类型与通用常量
packages/db             数据访问与迁移
packages/openai-client  OpenAI SDK 封装
packages/prompt-compiler Prompt 编译与 Patch 处理
packages/review-engine  自动评审调用与结果标准化
packages/job-runner     BullMQ 任务处理器
docs                    文档
db                      SQL schema / migration
```

## 2. 分层设计
### 2.1 前端层
负责：
- 页面渲染
- 任务创建表单
- 图片对比交互
- 人工决策提交

### 2.2 API 层
负责：
- CRUD 接口
- 发起任务
- 查询任务状态
- 提交人工决策
- 获取角色/图片/评审聚合数据

### 2.3 领域服务层
负责：
- Prompt 编译
- Prompt Patch 合成
- 自动评分标准化
- 状态流转校验

### 2.4 模型调用层
负责：
- 调用 Image API 进行候选图生成
- 调用 Responses API 进行视觉评审
- 调用 Responses API 进行精修编辑
- 统一错误处理与日志

### 2.5 队列层
负责：
- generation job 入队
- image review 入队
- patch generation 入队
- retry / dead-letter 处理

### 2.6 数据与文件层
负责：
- PostgreSQL 存业务数据
- S3/R2 存原图、缩略图、导出图

## 3. 推荐模块职责
### 3.1 prompt-compiler
输入：
- Universe 全局模板
- Character 模板
- Task 参数
- Variant 策略
- Negative 模板

输出：
- compiled_prompt
- compiled_negative_prompt
- prompt_debug_payload

函数建议：
- compilePrompt()
- buildVariantPrompts()
- applyPromptPatch()
- diffPromptVersions()

### 3.2 openai-client
封装：
- generateImages()
- editImage()
- reviewImage()
- createPatchProposal()

要求：
- 所有模型调用必须从这里发出
- 内置 request id、重试、日志、限流控制
- 不允许业务代码直接散落写 SDK 调用

### 3.3 review-engine
负责：
- 构建自动评审输入
- 调用 Structured Output schema
- 解析与校验评审结果
- 生成排名与建议动作

函数建议：
- reviewCandidateImage()
- rankReviewResults()
- summarizeFailureTags()

### 3.4 job-runner
负责：
- ExploreJobProcessor
- RefineJobProcessor
- ReviewJobProcessor
- PatchJobProcessor

状态：
- queued
- running
- reviewing
- completed
- failed

## 4. 核心业务状态流转
### 4.1 Generation Job
- queued -> running -> reviewing -> completed
- queued -> running -> failed
- reviewing -> failed

### 4.2 Generated Image
- created -> reviewed -> selected / rejected / candidate_master -> master

### 4.3 Anchor Image
- inactive -> active_master
- active_master -> deprecated_master

## 5. 生图策略
### 5.1 Explore 模式
目标：
- 在不确定最终最优表达时，受控试错

实现：
- 3 个 Prompt 变体
- 每个变体 4 张
- 共 12 张

推荐变体：
- ratio_boost：强化头身比
- style_lock：强化世界观风格锚点
- pose_clarity：强化动作与道具

### 5.2 Refine 模式
目标：
- 基于已有候选图修局部问题

要求：
- 必须指定基图 image_id
- 只修局部，不重写角色身份
- patch 内容必须保存

## 6. 自动评审架构
### 6.1 输入
- 候选图
- 世界观设定摘要
- 角色设定摘要
- 本次任务设定
- 评审规则
- JSON Schema

### 6.2 输出
- 总分
- 各维度分数
- 失败标签
- 修复建议
- best_use 建议

### 6.3 设计要求
- schema 校验失败视为任务失败
- 自动评审必须幂等
- 支持后续批量重评

## 8. 非功能要求
### 8.1 可追溯性
任意图片必须能反查：
- 来源任务
- Prompt 版本
- 评审结果
- 决策结果

### 8.2 幂等性
重复提交任务时避免重复插入；可通过 idempotency key 或任务签名实现。

### 8.3 可回滚性
角色标准稿切换必须有历史记录。

### 8.4 可观测性
必须记录：
- request id
- provider/model
- latency
- token/image usage
- error code
- queue retry count

## 9. 安全与约束
- 服务端保存 API key，前端绝不直连模型
- 上传图仅允许白名单 MIME
- 对存储对象使用签名 URL
- 记录关键操作审计日志
