# 08_提交前CI校验流程

## 目标
- 本地提交前执行与 GitHub Actions 一致的校验流程
- 避免“本地能提交，远端 CI 才报错”的往返成本

## 当前统一入口
- 根脚本：`pnpm ci:verify`
- 具体步骤：
  - `pnpm ci:lint`
  - `pnpm ci:typecheck`
  - `pnpm ci:test`
  - `pnpm ci:build`

## Git Hook
- 仓库使用版本化 hook：`.githooks/pre-commit`
- 提交前会自动执行：`pnpm ci:verify`
- 如果本机没有 `pnpm`，hook 会直接阻止提交并提示先安装 Node.js / pnpm

## 与 GitHub Actions 对齐
- `.github/workflows/ci.yml` 不再直接写散落命令
- workflow 改为调用同一组 `ci:*` 脚本
- 这样本地 hook 和远端 CI 共用同一套命令入口，减少漂移

## 注意事项
- 这套流程会比普通 pre-commit 更重，因为会跑完整 lint、typecheck、test、build
- 这是有意设计，用来保证“提交前必过一遍 CI 主流程”
