# AGENTS.md

## 项目名称
角色库工作台 / Character Factory

## 仓库目标
本仓库用于开发一个角色资产生产工作台，核心闭环为：
角色设定管理 -> Prompt 编译 -> 批量生图 -> 自动评审 -> 多图对比 -> Prompt Patch -> 标准稿锁定

## 工作方式
1. 先输出实施计划，再开始写代码。
2. 每次只解决一个 Sprint 或一个明确目标。
3. 所有改动必须与当前目标直接相关，不要无关扩展。
4. 改代码前先阅读 `docs/` 下对应文档。
5. 改动完成后输出：
   - 修改了哪些文件
   - 为什么这样改
   - 还有哪些风险点

## 编码约束
### 结构
- 使用 monorepo 结构
- 业务逻辑放 `packages/`
- 页面逻辑放 `apps/web`
- 不要在页面中直接写复杂业务逻辑
- 不要在页面组件里直接调用模型 SDK

### 类型与边界
- 全程使用 TypeScript
- 不要使用 `any`，除非明确说明原因
- 所有外部 API 数据都要经过 schema 校验
- 所有数据库写入都必须通过统一 repository/service

### 模型调用
- 所有 OpenAI 调用统一放在 `packages/openai-client`
- 不允许各处散落 SDK 调用
- 任何模型响应都必须有错误处理、日志和 request id
- 自动评审必须使用结构化输出 schema

### 数据与状态
- 所有图片必须绑定任务与 prompt 版本
- 所有 Master 图切换必须可回滚
- 所有 Patch 必须落库
- 任何状态迁移都要走统一状态机或 service 层

### 前端
- 组件尽量小而明确
- 先保证功能正确，再做视觉优化
- UI 以工作台效率为优先，不追求过度包装
- 关键页面必须支持 loading / error / empty 状态

### 测试与验证
- 完成功能后至少补最基本的单元测试或集成测试
- 提交前运行：
  - lint
  - typecheck
  - 核心测试
- 不要声称“已完成”但没有可验证结果

## 当前优先级
优先顺序固定为：
1. 基础工程
2. 角色与世界观 CRUD
3. Prompt 编译
4. Explore 批量出图
5. 自动评审
6. 多图对比
7. Refine 与 Patch 闭环

## 禁止事项
- 不要擅自更换技术栈
- 不要提前做视频生成
- 不要直接接入复杂权限系统
- 不要实现全自动定稿
- 不要引入与 MVP 无关的大型依赖

## 输出格式偏好
每次回复尽量包含：
1. 实施计划
2. 代码改动摘要
3. 验证结果
4. 下一步建议（仅 1 个）
